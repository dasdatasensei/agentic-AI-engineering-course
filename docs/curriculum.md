**Production note (5.6 — State compaction):** Long-running graphs accumulate message history that eventually exceeds the model's context window, degrading quality and inflating cost. Teach `trim_messages` (a non-destructive copy right before the LLM call) vs. `RemoveMessage` / `REMOVE_ALL_MESSAGES` (which edits the persisted checkpoint for good), and the summarise-then-remove pattern as a dedicated compaction node ahead of `checkpoint`. Authoritative sources: 📘 [LangChain — Short-Term Memory: trimming, deleting &amp; summarising messages](https://docs.langchain.com/oss/python/langchain/short-term-memory) · 📘 [LangGraph — `RemoveMessage` &amp; the `add_messages` reducer (delete-messages how-to)](https://docs.langchain.com/oss/python/langgraph/overview) · ⚠️ Caveat to teach: after trimming, the surviving history must remain provider-valid (start on a human message; every tool call must keep its matching tool result).

**Production note (6.5 — Resilient concurrency):** A bare `await asyncio.gather(*tasks)` cancels every sibling coroutine the instant one raises, so a single failing specialist takes down the whole supervisor turn. Teach `asyncio.gather(*tasks, return_exceptions=True)` so failures come back as values to be inspected, then a reduction step that separates successful `AgentResult`s from exceptions and lets the orchestrator decide whether to proceed on partial evidence, retry the failed agent, or degrade gracefully. Authoritative sources: 📘 [Python `asyncio` — Running Tasks Concurrently (`gather`, `return_exceptions`)](https://docs.python.org/3/library/asyncio-task.html#asyncio.gather) · 📘 [Python `asyncio` — Timeouts (`asyncio.wait_for`, `asyncio.timeout`)](https://docs.python.org/3/library/asyncio-task.html#timeouts) · 📘 [LangGraph — durable execution &amp; error recovery for parallel nodes](https://docs.langchain.com/oss/python/langgraph/overview).
